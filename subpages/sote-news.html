Introduction

State of the Earth (sote) News is an innovative news aggregation platform designed to automatically collect, process, and analyze environmental news articles from multiple sources. It produces summarized insights, categorized topics, and visual media based on each article, leveraging AI technologies and serverless architecture.
In the face of growing environmental concerns, staying informed about the planet’s condition is crucial. By quickly providing concise, curated information on environmental topics, this platform empowers users to stay informed on vital issues without requiring extensive time investments.

Project Goals

  • Automate ingestion of environmental news articles: Set up a fully automated pipeline to continuously collect and ingest news articles from various environmental sources.
  • Apply machine learning models for summarization and topic generation: Use machine learning models to generate concise summaries and automatically classify articles by relevant topics.
  • Enhance articles with AI-generated images: Add AI-generated images to each article to visually represent key themes and engage readers.
  • Prepare processed data for analytics in a Redshift data warehouse: Organize the processed data within a Redshift data warehouse to enable efficient analytics and reporting.





Key Features

  • Automated News Scraping: Periodically scrapes environmental articles from various RSS feeds
  • AI-Powered Summarization and Topic Generation: Utilizes OpenAI's GPT and Hugging Face's BART models to generate summaries and topics.
  • AI-Generated Images: Utilizes Stability AI's Stable Diffusion model to create visual representations of article summaries
  • Redshift Integration: Enables analytical querying and reporting
  • Serverless Architecture: Leverages AWS Lambda, S3, and Redshift for scalability
  • Event-Triggered Pipeline: Maintains an efficient and automated workflow with event-based triggers.

Workflow

This project operates as a multi-step data pipeline, ingesting environmental news articles, processing them with AI tools, and uploading them to AWS Redshift for analytical purposes.

Workflow Summary

  • Lambda 1: Scrapes news articles from RSS feeds → stores raw CSV in 1_raw/ folder
  • Lambda 2: Summarizes and categorizes articles → stores summarized CSV in 2_summarized_with_topics/
  • Lambda 3: Generates images based on summaries → stores CSV with image links in 3_generated_images/
  • Lambda 4: Inserts data into Redshift

Detailed Workflow

Lambda Function 1: Scraping News Articles

  • Purpose: Periodically scrape environmental news articles from various RSS feeds and store the raw content in a structured format (CSV) in S3.
  • Input: Scheduled trigger, RSS feeds
  • Process: Scrape articles, structure data into CSV
  • Output: Raw CSV in 1_raw/ S3 folder

Lambda Function 2: Summarize Articles and Generate Topics

  • Purpose: Process raw articles, summarize content, and classify into predefined topics using AI models.
  • Input: Raw CSV from 1_raw/ S3 folder
  • Process: Summarize with BART, generate topics with GPT
  • Output: Summarized CSV in 2_summarized_with_topics/ S3 folder

Lambda Function 3: Generate Images

  • Purpose: Create images that visually represent the article summaries using AI-based image generation.
  • Input: Summarized CSV from 2_summarized_with_topics/ S3 folder
  • Process: Generate images based on summaries and titles
  • Output: CSV with image links in 3_generated_images/ S3 folder

Lambda Function 4: Insert Data into Redshift

  • Purpose: Insert processed data into AWS Redshift data warehouse for analysis.
  • Input: CSV from 3.1_generated_images/ S3 folder
  • Process: Generate and execute SQL COPY command
  • Output: Data loaded into Redshift table

Streamlit Application: Manual Workflow Control

  • Purpose: Provide user interface for manual triggering and monitoring of pipeline steps.
  • Features: CSV upload, Lambda function triggering, data review, file download





Deployment

AWS Lambda Functions

  • lambda_1_scrapeNewsArticles: Scheduled trigger
  • lambda_2_summarizeAndGenerateTopics: Triggered by S3 upload to 1_raw/
  • lambda_3_generateImages: Triggered by S3 upload to 2_summarized_with_topics/
  • lambda_4_insertRedshift: Triggered by S3 upload to 3.1_generated_images/

AWS Lightsail Hosting

  • Flask App Deployment: The Flask application is deployed to AWS Lightsail for reliable and scalable hosting.

VPC Configuration

  • Secure AWS Services: All AWS services reside within a VPC and are not publicly accessible.
  • Lightsail VPC Peering: The Lightsail instance resides in a separate VPC and communicates securely with Redshift via VPC peering.
